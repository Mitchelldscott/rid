pub fn init(&mut self, header: &[u8], data: &[u8])

Copy the TaskNode header data from a buffer to Self

If the driver in the header changes the TaskNode will start configuring for that driver immediately. Called when a client recieves an init packet. Only clients should recieve init packets.

Also sets the TaskExecutable to the new driver (if it is a new driver).

source

pub fn header(&self) -> (TaskHeader, TaskBuffer)

Used to share the current header of a task. This is only useful on a host sending an init packet.

source

pub fn collect_chunk(&mut self, header: &[u8], data: &[u8])

Collect a packet containing config data. The data is only consumed if the TaskNode is already in the TaskStatus::Configuration state.

source

pub fn kill(&mut self)

Resets all task data and enters TaskStatus::Standby state.

source

pub fn configure(&mut self)

Configure the task. Activates the task if all chunks have been collected. If the TaskExecutable is not initialized there was an error initializing and the node reached this state in error, revert to standby.
